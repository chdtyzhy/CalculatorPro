# 版本管理

## 🎯 版本策略

### 版本号格式
```
主版本号.次版本号.修订号
例: v1.2.3
```

### 版本号含义
```
主版本号 (1): 重大更新，不向下兼容
次版本号 (2): 功能更新，向下兼容
修订号 (3): Bug修复，向下兼容
```

### 预发布版本
```
v1.0.0-alpha.1    # Alpha测试版
v1.0.0-beta.1     # Beta测试版
v1.0.0-rc.1       # 发布候选版
v1.0.0            # 正式发布版
```

## 🚀 发布流程

### 1. 开发阶段 (dev分支)
```
✅ 功能开发
✅ 单元测试
✅ 集成测试
✅ 用户体验测试
```

### 2. 测试阶段 (release分支)
```
✅ 创建release分支

### 3. 发布阶段 (main分支 + 标签)
```

## 📋 版本控制

### Git标签
```bash
# 创建标签
git tag -a v1.0.0 -m "版本 1.0.0"

# 推送标签
git push origin --tags

# 查看标签
git tag -l

# 查看标签详情
git show v1.0.0
```

### GitHub Release
```
1. 在GitHub仓库创建Release
2. 关联Git标签
3. 添加发布说明
4. 上传二进制文件 (如果适用)
5. 发布到GitHub
```

## 🔧 版本兼容性

### 向后兼容保证
```
✅ v1.x.x: 保证API向后兼容
✅ 重大更新: 仅在主版本号变更时
✅ 数据迁移: 提供迁移工具和指南
✅ 用户通知: 重大变更提前通知
```

### 支持周期
```
🔄 当前版本: 完全支持
🔄 上一个主版本: 安全更新支持
🔄 更早版本: 社区支持
```
